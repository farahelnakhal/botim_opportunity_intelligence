# knowledge-base/product-ideas/

Owned by Workstream B (Product & Opportunity Intelligence).

Holds one file per product proposition, plus the living backlog.

## Contents

- `BACKLOG.md` — the product-opportunity backlog (create from `opportunity-intelligence/templates/opportunity-backlog.md` when the first real proposition is logged).
- `<idea-slug>.md` — opportunity profile per proposition: opportunity-framework fields, value proposition, stress test, classification, MVP section, links to scorecard/model/experiments.
- `<idea-slug>-recommendation-YYYY-MM-DD.md` — issued meeting-ready outputs.

## Rules

- Update existing entries rather than restarting research; keep a changelog line per material change.
- Rejected ideas are archived in the backlog with a reopen trigger — never deleted.
- Evidence is cited by ID from `knowledge-base/customer-evidence/` (read-only — Workstream A owns it). Missing evidence is marked `(A)` and logged in the backlog's evidence-request queue.
