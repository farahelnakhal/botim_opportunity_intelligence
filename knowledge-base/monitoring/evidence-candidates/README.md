# Evidence Candidates — Workstream A's intake

External detections that may constitute customer/market evidence land here (one markdown file per candidate: facts, sources, access labels, fetch dates, the triggering EVT id). **Workstream A** reviews, validates under its own rules (authenticity screen, injection defense, strength ladder), and either promotes to an EV record (noting the candidate path) or rejects with a one-line reason appended to the candidate file. Workstream C never writes EV records directly.
