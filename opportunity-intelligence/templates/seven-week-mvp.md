# Seven-Week MVP Template

Define an MVP that can genuinely ship (or convincingly simulate) in seven weeks. If the honest answer requires licences, scheme onboarding, or integrations that take longer, say so and design a concierge/manual MVP instead — do not pretend.

## 1. Scope

- **Target segment:** narrow deliberately (one segment, one city if needed).
- **Product concept:** one paragraph.
- **Main customer journey:** onboarding → first use → repeat use, in ≤8 steps.
- **Payment mechanism / credit mechanism / repayment mechanism:** what is real, what is manual behind the scenes, what is out of scope.
- **Revenue model in MVP:** what is actually charged (it is acceptable to charge nothing IF the MVP still tests willingness to pay another way — say how).
- **Switching trigger targeted:** which inflection point recruitment will exploit.

## 2. Feature cut

| Essential (MVP fails without) | Non-essential (explicitly deferred) |
|---|---|
| | |

Rule: if a feature doesn't test the core hypothesis or enable the journey, it is non-essential.

## 4. Dependencies

- **Required data:** (incl. requests to Customer & Market Intelligence module)
- **Required integrations:** (KYB, issuer-processor, BIN sponsor, payment rails, core wallet)
- **Required partners:** and status (confirmed / in discussion / unidentified)
- **Regulatory constraints:** what the MVP can do without additional licensing, and the compliant workaround used (e.g. AstraTech lends on its own licence; wallet under existing e-money arrangement).

## 5. Metrics

| Success metrics (targets) | Failure metrics (kill thresholds) |
|---|---|
| e.g. ≥X merchants onboarded, ≥Y% route ≥Z% of spend, repayment ≥W% | e.g. <X merchants despite N offers, routing <Y%, drawdown but no routing |

Both columns are mandatory. An MVP without a kill threshold is a commitment, not a test.

## 6. Expansion path

If success metrics hit: the next segment, feature, or geography — one paragraph, no roadmap theatre.

Store in `knowledge-base/product-ideas/<idea-slug>.md` (MVP section) or a linked file.
