# Product-Opportunity Backlog Template

Single living document: `knowledge-base/product-ideas/BACKLOG.md`. Update it whenever a proposition is added, re-scored, re-classified, or archived. Never delete rows — move them to the archive section so rejected ideas aren't re-litigated from scratch.

## Row rules

- **ID:** OPP-### assigned once, never reused.
- **Classification:** only from the stress-test framework; a classification change requires a note in the changelog below.
- **Composite:** shown to one decimal, always labelled indicative; the scorecard link carries the real information.
- **Next action:** exactly one — an experiment ID, an evidence request to Customer & Market Intelligence, a modelling task, or "archive".
- Each row links to: opportunity profile (`product-ideas/`), scorecard (`opportunity-scores/`), commercial model (`commercial-models/`), experiments (`validation/`).

## Ordering

Sort by classification (Strong → Promising → Weak), then by ease of validation. Rejects live in the archive.
